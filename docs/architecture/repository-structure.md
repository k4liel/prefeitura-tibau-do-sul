# Estrutura do repositorio

## Pastas principais

- `backend/` - sistema Django modular (desenvolvimento ativo)
- `docs/` - documentacao funcional/tecnica
- `data/` - exports consolidados para conferencia e bootstrap
- `legacy/` - artefatos da fase estatic–∞ inicial
- `skills/` - playbooks de execucao do time
- `scripts/` - scripts utilitarios do legado

## Convencao de nomes adotada

- arquivos markdown em `kebab-case`
- nomes sem acentos e sem espacos
- separacao clara por contexto: `architecture/`, `plans/`, `checklists/`, `research/`

## Regras de organizacao

1. codigo ativo novo entra em `backend/`
2. material historico vai para `legacy/`
3. evidencias e pesquisas ficam em `docs/research/`
4. checklists operacionais ficam em `docs/checklists/`
